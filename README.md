# douyin review skills

抖音创作者后台**周度数据分析复盘**技能。把一周的运营数据（Excel 表 + 后台截图）丢给它，自动按既定分析框架生成一份可视化 HTML 复盘报告。

> 适合：抖音短视频账号运营者、新媒体团队，做每周数据复盘 / 周报 / 归因分析。

---

## 功能概述

- **一键周报**：提供周维度数据，自动产出结构化 HTML 复盘报告，无需手动拼表格。
- **九大分析维度**：数据下降归因、上周数据总览、近 7 日关注来源、重点数据关注、单条视频深度拆解、亮点数据、待优化指标、后续优化策略、重点工作方向。
- **客观 + 积极**：归因不堆数据、不制造焦虑，每条结论往建设性方向引导。
- **爆款可复制**：单条视频拆解沉淀"值得学习"与"需优化"两栏，把偶然爆款提炼成可复用模型。
- **可视化**：内置 Chart.js 交互图表（趋势图、关注来源占比等）。

### 它**不**做什么

- 不登录抖音后台抓数据（后台需扫码/验证码，本环境无浏览器自动化能力）。
- 不碰"掉粉 / 净增粉丝 / 取关"做归因（这是写进技能红线的基本原则，详见下方注意事项）。
- 不联网上传你的任何数据，全部本地处理。

## 快速开始

1. 打开抖音创作者后台，导出（或截图）本周的运营数据：
   - Excel：`数据表现_播放量数据` `作品点赞数据` `作品评论数据` `作品分享数据` `主页访问数据` `封面点击率数据` 等。
   - 截图：近 7 日**关注来源**、**单条视频**互动 / 完播数据、粉丝画像（可选）。
2. 在 WorkBuddy 对话里把这些文件 / 截图一次性发过来，并说：
   > "帮我做上周的数据复盘"
3. 技能会自动解析数据 → 生成 HTML 报告 → 在预览面板打开，并附一段核心结论摘要。

## 使用说明

### 输入数据要求

| 类型 | 内容 | 用途 |
|---|---|---|
| Excel 表（每日一行） | 播放量、点赞、评论、分享、主页访问、封面点击率、净增粉丝、取关、总粉丝量 | 趋势图、KPI 概览、归因 |
| Excel 表 | 单条视频明细（标题、发布时间、播放、点赞、评论、转发、完播率 / 人均时长、涨粉、2s 跳出率） | 单条视频拆解 |
| 截图 | 近 7 日关注来源及占比 | 关注来源板块 |
| 截图（可选） | 粉丝画像（性别 / 年龄 / 地域） | 补充解读 |

- 数据缺某维度时，对应板块会标注"待补充"，**不会编造**。
- 有前一周数据可算环比；没有则只报当前值。

### 触发方式

- 自然语言：发数据 + "做周报 / 数据分析复盘 / 数据归因"。
- 显式调用：`@skill:douyin-review-skills` 后附数据。

### 执行流程（技能内部）

1. 读数据：Node `xlsx` 解析 Excel，`Read` 直接读截图取数。
2. 算指标：周合计、日环比、关键变幅（峰值、周末跌幅、渠道增幅）。
3. 写报告：单文件 HTML，内联 CSS，Chart.js CDN 画图。
4. 交付：`present_files` 打开预览 + 文字结论摘要。

## 输出文档结构

生成的报告为**单个 HTML 文件**，板块顺序固定如下：

1. **上周核心数据总览** — KPI 卡（播放 / 点赞 / 评论 / 分享 / 主页访问 / 封面 CTR 周合计 + 环比）+ 趋势图（播放 / 点赞 / 分享 / 主页访问四条线，**不含净增粉丝**）+ 「上周需重点关注的数据维度」6 张关注卡。
2. **数据下降归因** — 围绕内容供给 / 互动设计 / 封面标准化的客观归因，**不出现掉粉维度**，每条结尾积极引导。
3. **近 7 日关注来源** — 来源占比 + 亮点 / 萎缩渠道解读。
4. **重点数据关注** — 3–5 个最该盯住的信号。
5. **单条视频数据分析** — 每条视频数据 + 左右两栏「值得学习和沉淀 / 可优化提升」（严重问题标红「必须修改提升」）。
6. **亮点数据** — 本周正向信号。
7. **待优化指标** — 偏弱、下周要改的指标。
8. **后续优化策略** — 可落地动作。
9. **重点工作方向** — 下周 2–3 个最高优先级。

## 性能与最低硬件配置

本技能计算量极轻（解析若干 Excel + 生成静态 HTML），对硬件几乎没有要求：

| 项目 | 最低要求 | 说明 |
|---|---|---|
| CPU | 任意双核 x86 / ARM | 解析在秒级完成 |
| 内存 | ≥ 4 GB | 富余，数据量通常 < 1 万行 |
| 磁盘 | 几 MB 空闲 | 仅存技能与输出 HTML |
| 运行时 | WorkBuddy 内置 Node 22 / Python 3.13 隔离环境 | 也可本地 Node ≥ 18、Python ≥ 3.9 |
| 网络 | 打开报告时需联网 | Chart.js 走 CDN 加载图表；数据解析本身不联网 |

**典型耗时**：9 个 Excel + 7 张截图，解析到生成 HTML 通常 **< 10 秒**（取决于数据规模）。

## 注意事项

### 红线清单（技能强制遵守，改动需谨慎）

1. **板块二不碰掉粉**：归因里不出现"掉粉 / 净增粉丝 / 取关"字眼，数据表也不放这两列。
2. **趋势图不含净增粉丝**：板块一趋势图只放播放 / 点赞 / 分享 / 主页访问。
3. **不堆数据**：每个数据都配"为什么"（归因 / 客观判断）。
4. **客观 + 积极引导**：尤其归因板块，结论往建设性收。
5. **单条视频双栏分析**：数据 + 「值得沉淀 / 需提升」，严重问题标红。
6. **去 AI 味**：避免破折号堆砌、机械加粗、排比三连、促销腔。

### 数据与隐私

- 所有数据**仅在本地处理**，不会上传任何服务器。
- 抖音后台需登录才能进，技能**不会**尝试代登录；请自己导出 / 截图后提供。

### 其他

- 报告样例基准见同目录 `SKILL.md` 引用的 `抖音周度数据分析报告_0706-0712.html`。
- 改标准只需编辑 `SKILL.md`；红线措辞调整请同步更新本文档。
