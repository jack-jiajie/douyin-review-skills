# 抖音周度数据分析复盘 v3

当用户甩过来一周的抖音后台数据（Excel 表 + 截图），并要求做数据分析复盘时，按本 skill 的标准输出一份**可视化 HTML 复盘报告**。

本 skill v3 相比 v1/v2 有三处根本性升级，请务必理解其设计意图，而不是只套模板：

1. **先校准，后分析（防错前置）**——一次性能收到十几个文件/截图，文件名会误导、截图会混用。v3 要求**先**把"每个指标到底来自哪个文件/截图、数值是多少"理清楚、让用户确认，**再**出分析，从源头杜绝"把点赞量当成评论量一直在分析"这类张冠李戴。
2. **三位一体复盘（拒绝数据罗列）**——单条视频不再只摆数字，而是「数据反馈结果 + 视频过程指标 + 视频内容」三者交叉，并追问为什么好/不好、可复用什么、可沉淀什么、是否形成闭环。
3. **结论先行（运营周报逻辑）**——领导没空看 9 个罗列板块。报告开头先给"一句话总评 + 3 个关键发现 + 3 件必做"，正文只保留真正驱动数据走向的因素，砍掉纯数据堆砌。

## 一、触发场景

- 用户提供周维度的抖音运营数据（文件名常含「数据表现_播放量数据 / 作品点赞数据 / 作品评论数据 / 作品分享数据 / 主页访问数据 / 净增粉丝数据 / 取关粉丝数据 / 总粉丝量数据 / 封面点击率数据」等 Excel，外加关注来源、单条视频、粉丝画像、直播引流等截图）。
- 用户说"做周报""数据分析复盘""数据归因""上周数据怎么样""对上周分析复盘"等。
- 用户明确要求"按之前的标准来"或引用本 skill。

> 抖音后台必须登录才能进，本环境无浏览器自动化能力，**不要尝试自己登录后台抓数据**。若用户只给了链接没给数据，先请对方把 Excel 表和截图发过来（参考下方所需字段）。

---

## 二、整体工作流（v3：两阶段，阶段0不可跳过）

```
阶段0  资料整理与校准  ──→  产出「指标真相表」→ 请用户确认 → 这是后续所有分析的取数依据
  │
  ▼
阶段1  报告生成        ──→  结论先行 + 精简总览 + 三位一体复盘 + 闭环策略
```

**阶段0 是硬性前置门禁**：未拿到用户对「指标真相表」的确认前，不得进入任何分析撰写。这是 v3 最重要的防错机制。

各阶段的具体操作、报告板块定义、图表规范、红线、写作标准、OCR 实操均已拆为独立文件，存于 `references/`。执行时按需要 `Read` 对应文件：

| 功能模块 | 文件 | 用途 |
|----------|------|------|
| 阶段0 资料校准 | `references/00-calibration.md` | 文件识别 + 截图 OCR 归类 + 指标真相表闸门 |
| 报告板块结构（板块一~八） | `references/01-report-structure.md` | 8 大板块的定义、目的与写法 |
| 单条视频三位一体框架 | `references/02-three-in-one.md` | 板块四详解：数据+过程+内容+为什么+复用+沉淀+闭环 |
| 图表设计规范 | `references/03-chart-design.md` | 趋势图双 Y 轴/分面、可观性与有效性自检 |
| 红线清单 | `references/04-red-lines.md` | 10 条不可违反的硬性规则 |
| 去 AI 味写作标准 | `references/05-deai-style.md` | humanizer 集成、禁用清单、自查 |
| 截图 OCR 实操 | `references/06-ocr-guide.md` | 腾讯云高精度 OCR（首选）+ 本地 tesseract 兜底 |
| 输出模板与交付话术 | `references/07-output-template.md` | HTML/CSS/Chart.js 骨架 + 给用户的汇报话术 |

**推荐执行顺序**：先读 `00-calibration.md` 做阶段0 → 用户确认后读 `01-report-structure.md` + `02-three-in-one.md` 写正文 → 画图表前读 `03-chart-design.md` → 成稿前读 `05-deai-style.md` 过机器腔 → 写图识别有问题时读 `06-ocr-guide.md`。

## 三、红线总览（逐条细则见 `references/04-red-lines.md`）

1. 阶段0 校准不可跳过；易混指标（点赞/评论/作品搜索/账号搜索/净增粉丝/总粉丝）必须逐行核对。
2. 全报告不碰"掉粉 / 净增粉丝 / 取关"归因（净增粉丝仅作客观卡呈现）。
3. 趋势图不含净增粉丝，只放播放/点赞/分享/主页访问。
4. 不堆裸数据，每个数字配"为什么"。
5. 客观 + 积极引导收尾。
6. 单条视频必须有三位一体 ④⑤⑥⑦ 深层复盘，严重问题标红。
7. 结论先行（板块一必须是结论速览）。
8. 去 AI 味（强制过 humanizer 标准）。
9. 亮点/待优化/优化策略三板块定位不可混、板块七↔板块八一一对应、均不退化成裸罗列。
10. 截图识别优先腾讯云高精度 OCR，仅在无密钥/断网时回退本地 tesseract。

> 任一红线被违反，报告即视为不达标，必须改到全部满足再交付。
