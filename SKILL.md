# 抖音周度数据分析复盘

当用户甩过来一周的抖音后台数据（Excel 表 + 截图），并要求做数据分析复盘时，按本 skill 的标准输出一份**可视化 HTML 复盘报告**。

本 skill 的核心价值不是"读数据"，而是**按已和用户对齐的分析框架与表达标准**出报告——尤其是几条反复调过的硬性规则（见下方「红线清单」），必须严格遵守。

---

## 一、触发场景

- 用户提供周维度的抖音运营数据（文件名常含「数据表现_播放量数据 / 作品点赞数据 / 作品评论数据 / 作品分享数据 / 主页访问数据 / 净增粉丝数据 / 取关粉丝数据 / 总粉丝量数据 / 封面点击率数据」等 Excel，外加关注来源、单条视频、粉丝画像等截图）。
- 用户说"做周报""数据分析复盘""数据归因""上周数据怎么样"等。
- 用户明确要求"按之前的标准来"。

> 抖音后台必须登录才能进，本环境无浏览器自动化能力，**不要尝试自己登录后台抓数据**。若用户只给了链接没给数据，先请对方把 Excel 表和截图发过来（参考下方所需字段）。

---

## 二、所需输入字段（拿到数据先核对）

**Excel 数据表（每日一行，列是日期）**
- 播放量、点赞量、评论量、作品分享量、主页访问量、封面点击率
- 净增粉丝、取关粉丝、总粉丝量（用于数据总览 KPI，但**不进趋势图、不进下降归因**）
- 单条视频明细（标题、发布时间、播放、点赞、评论、转发、完播率/人均时长、涨粉数、2s 跳出率）

**截图（用户通常直接发图，用 Read 读图取数）**
- 近7日关注来源及占比（视频推荐 / 我的主页 / 搜索 / 关注页 / 直播页 / 其他）
- 单条视频的互动与完播数据
- 粉丝画像（性别 / 年龄 / 地域，可选）

若数据缺失关键维度（如没有单条视频明细），先出已有部分，并在对应板块注明"待补充"，不要编造。

---

## 三、执行步骤

1. **读数据**
   - Excel 解析：`Bash` 用 Node `xlsx` 库（`NODE_PATH=/Users/xdf/.workbuddy/binaries/node/workspace/node_modules /Users/xdf/.workbuddy/binaries/node/versions/22.22.2/bin/node -e "..."`）或 Python `openpyxl`。优先 Node xlsx，环境已装。
   - 截图：`Read` 工具直接读图取数。
   - 把每日数据整理成结构化数组（日期、播放、点赞、评论、分享、主页访问、封面CTR、净增粉、取关等）。

2. **算指标**
   - 周合计、日环比、关键变幅（如某日播放峰值、周末跌幅、推荐渠道增幅）。
   - 关注来源占比与上周对比（有前周数据才算环比，没有就只报当前占比）。

3. **写 HTML 报告**
   - 单文件 HTML，内联 CSS，图表用 Chart.js CDN（`https://cdn.jsdelivr.net/npm/chart.js`）。
   - 严格按下方「报告板块结构」组织，遵守「红线清单」。
   - 语言：简体中文；风格：去 AI 味（参考 `humanizer` / `去AI味` skill 的禁忌清单）。

4. **交付**
   - `present_files` 打开 HTML 预览，并附一段核心结论摘要（不要只说"报告已生成"，要给出 2-3 条最重要的信号）。

---

## 四、报告板块结构（顺序固定，共九大块）

### 板块一：上周核心数据总览
- **KPI 概览卡**：播放、点赞、评论、分享、主页访问、封面 CTR 的周合计 + 环比（如有前周）。可保留净增粉丝作为客观数据卡，但仅作呈现、不展开归因。
- **趋势图（Chart.js）四条线**：播放量、点赞、分享、主页访问。
  - ⚠️ **红线**：趋势图**不含净增粉丝**系列；右轴标题为"互动量 / 主页访问"，不出现"粉丝"字样；tooltip 里不得残留 `x10` 取值分支。
- **【必含】上周需重点关注的数据维度**：趋势图下方放 6 张关注卡，每张含「维度 / 当前信号 / 重点关注什么 / 状态标签（亮点绿、问题红橙）」：
  1. 播放量稳定性（倒V型？爆款后有无同质量内容接住）
  2. 评论互动率（是否埋了评论钩子 + 运营前 30 分钟）
  3. 封面点击率（能否沉淀可复用封面 SOP）
  4. 视频推荐占比（能否用同系列做连带观看、把单点推成矩阵）
  5. 主页 / 搜索渠道（IP 辨识度、标题关键词布局）
  6. 单条完播 / 首屏（新视频首屏 3 秒 + 完播率是否达标）

### 板块二：数据下降归因
- ⚠️ **红线（最高优先级）**：**绝对不出现"掉粉 / 净增粉丝 / 取关"维度**，连数据表都不放净增粉、取关两列。
- 围绕**内容运营**做客观归因，通常 3 条：
  1. 播放量"倒V型" → 归因"爆款之后接不住"（供给断层、发布时间错位、未矩阵化）
  2. 互动低迷 → 归因"内容在输出，没在引发"（缺评论钩子、情绪单向、黄金窗口未运营）
  3. 封面 CTR 过山车 → 归因"靠灵感不靠系统"（缺模板、信息密度不稳、图文协同弱）
- 每条归因结构：**现象 → 背后原因 → 客观判断**，结尾必须**往积极方向引导**（例："倒V型恰恰说明前半周内容质量过关""CTR 能冲到 27%+ 说明你完全具备吸睛能力"）。
- 配套数据表（已去掉净增粉 / 取关列）。

### 板块三：关键运营指标 —— 近7日关注来源
- 来源占比条形 / 列表（视频推荐、我的主页、搜索、关注页、直播页、其他）。
- 分析：哪个渠道是亮点（如视频推荐 +500%）、哪个在萎缩（如主页 -52%、搜索 -50%），并给客观解读，不恐慌化。

### 板块四：重点数据关注
- 挑 3-5 个最该被盯住的数据信号，简短说明为什么重要、当前状态、下周该看什么。

### 板块五：单条视频数据分析（深度拆解）
- ⚠️ **红线**：**不只罗列数据**，每条视频下方必须拆成左右两栏分析：
  - 左栏「**值得学习和沉淀**」（绿色）
  - 右栏「**可优化提升**」（橙色；表现严重翻车的用红色标题「**必须修改提升**」）
- 具体分析要素：标题钩子、首屏 3 秒（2s 跳出率）、完播率、情绪共鸣、实用/收藏价值、评论区激活、爆款是否趁热承接。
- 末尾「小结」：把几条视频统一到一个可复用模型（如"具体场景 + 情绪钩子 + 信息密度"），落到"工程化放大"的积极结论。

### 板块六：亮点数据
- 本周正向信号（如推荐渠道爆发、某条爆款、CTR 高点），用事实 + 数字说话。

### 板块七：待优化指标
- 本周偏弱、下周要重点改的指标，客观列举，不自我贬低。

### 板块八：后续优化策略
- 基于前面分析给可落地动作（如：复制爆款公式、每条视频埋评论钩子、沉淀封面 SOP、发布时间固定化、用同系列做矩阵）。

### 板块九：重点工作方向
- 下周 2-3 个优先级最高的方向，简洁收尾。

---

## 五、红线清单（务必逐条遵守）

1. **板块二不碰掉粉**：不出现"掉粉 / 净增粉丝 / 取关"字眼与数据，不因粉丝流失做归因。
2. **趋势图不含净增粉丝**：板块一趋势图只放播放/点赞/分享/主页访问。
3. **不堆数据**：每个数据都要配"为什么"（归因 / 客观判断），杜绝只甩数字的表格。
4. **客观 + 积极引导**：尤其板块二，归因结尾一律往建设性、积极面收。
5. **单条视频双栏分析**：数据 + 「值得沉淀 / 需提升」两栏，严重问题标红。
6. **去 AI 味**：参考 `humanizer` / `去AI味` skill——少用破折号、机械加粗、排比三连、促销腔；用具体细节替代空泛断言；中文用中文引号。
