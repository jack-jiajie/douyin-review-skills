## v3.2.1 · 2026-07-20 18:00 · 趋势图可读性优化

**触发**：用户指出当前趋势图（播放量 + 点赞 + 分享 + 主页访问 4 线同轴）因量级悬殊，点赞/分享/主页访问被压成底部直线，完全看不出趋势，图表失去意义。要求站在「可观性 + 有效性」角度重做。

**改动**：
- 板块二「核心数据总览」新增趋势图设计强制要求：
  - 播放量（千级）与点赞/分享/主页访问（十/百级）共用单轴会淹没小指标，必须改用**双 Y 轴**（左轴播放量、右轴互动 + 主页访问）或**分面小图**（上图播放量、下图互动 + 主页访问）。
  - 新增图表自检问题：除播放量外其他线是否真的起伏可见？若全部贴底则图表不合格。
  - 明确趋势图目的：让抽象数字下的走势和异常点一目了然，而非把所有数据堆在一张图里。
- 红线清单增加「趋势图有效性」条目：不得生成只有播放量一条线能看、其余指标贴底的废图。
- `README.md` 功能概述增加「趋势图真正可读」说明。

**同步落地**：当前报告 `抖音周度数据复盘_0713-0719_v3.html` 已改为双 Y 轴（左轴播放量蓝、右轴点赞绿/分享橙/主页访问紫）。

## v3.2.0 · 2026-07-20 17:33 · 集成去 AI 味 + 腾讯云 OCR

**触发**：用户要求在 skill 中加入「去 AI 味」与「腾讯云通用文字识别 OCR」两个能力，使撰写不带机器腔、截图识别更准。

**改动**：
- **去 AI 味**：阶段1 原则新增「语言去 AI 味」条目；红线第 8 条升级为「强制过 humanizer 标准」，列出禁用项（破折号 / 三连排比 / 促销腔 / 虚浮拔高 / AI 高频词），要求用具体细节替代空泛断言；参考区指明撰写后必须用 `humanizer` 技能过一遍。
- **腾讯云高精度 OCR**：阶段0 识别工具改为「优先腾讯云 `tencentcloud-ocr`、本地 tesseract 兜底」双方案；附录 A 重写为 A1（腾讯云，含 SDK 安装、密钥环境变量、`scripts/main.py --image-base64` 调用，返回 `raw_text`）+ A2（本地 tesseract 兜底，保留 BMP / 反相 / 关沙箱踩坑）；红线第 10 条明确「截图识别优先腾讯云 OCR」。
- `README.md` 同步增加两条特性说明。

**前置依赖**：腾讯云 OCR 需配置 `TENCENTCLOUD_SECRET_ID` / `TENCENTCLOUD_SECRET_KEY` 环境变量（购买「通用文字识别高精度版」）。无密钥时自动回退本地 tesseract，不影响流程。

## v3.1.0 · 2026-07-20 17:30 · 恢复亮点 / 待优化 / 优化策略三板块

**触发**：用户反馈需保留「亮点数据 / 待优化方向 / 优化策略」三个独立板块，并明确各自定位。

**改动**：
- 撤销 v3.0 把「亮点 / 待优化 / 策略 / 方向」合并为单块的决策，恢复为：
  - 板块六 亮点数据：让领导看账号上周**做对了什么**（成绩单视角）。
  - 板块七 待优化方向：客观指出**不足 / 短板**（不制造焦虑）。
  - 板块八 优化策略：针对不足给**一一对应的解决方案**（行动出口）。
- 总结构由 6 板块升为 **8 板块**；每个板块写明「目的 + 写法要求（拒绝数据罗列）」。
- 红线清单新增第 9 条：三板块定位不可混、板块七↔板块八必须一一对应、均不得退化成裸数据罗列。
- `README.md` 同步更新为 v3.1（成绩 / 不足 / 解法三段递进）。

## v3.0.0 · 2026-07-20 16:56 · 大版本重写（校准门禁 + 三位一体 + 结论先行）

**触发**：用户提出三点核心问题 + 一个新增需求：
1. 不要简单数据罗列，要结合「数据反馈结果 + 视频过程指标 + 视频内容」三位一体整体复盘。
2. 分析不要只说好坏，还要分析为什么好/不好、可复用、可沉淀、是否闭环等深层复盘。
3. 纯数据罗列板块太多、领导阅读成本高、抓不住重点、未提炼真正驱动数据走向的关键因素；要更符合运营周报逻辑。
4. 新增「整理文件和整理截图」前置功能——用户一次给很多截图/文件，需按截图文字内容 + 文件命名归纳，确保数据归属正确（点名本次曾把「点赞量当成评论量」分析）。

**改动**：
- **阶段0 资料整理与校准（硬性前置门禁）**：文件按「命名 + 表头双重识别」（防文件名误导），截图走 OCR 旁路归类；产出「指标真相表」请用户确认后才进分析；专门设「点赞 / 评论 / 作品搜索 / 账号搜索 / 净增粉丝 / 总粉丝」易混核对项，杜绝张冠李戴。
- **单条视频三位一体框架**：数据反馈结果 + 过程指标（完播 / 2s 跳出 / 5s 完播 / 时长 / CTR）+ 内容拆解（选题 / 钩子 / 结构 / 标题 / 封面），并强制 ④为什么好/不好 ⑤可复用 ⑥可沉淀 ⑦是否闭环 四步深层复盘。
- **结论先行 + 板块精简**：报告从 9 板块压缩为 6 板块（结论速览 / 精简总览 / 关注来源 + 关键指标 / 三位一体复盘 / 引流直播间 / 关键驱动因素与策略），开头即给「一句话总评 + 3 关键发现 + 3 必做」。
- 红线清单新增第 7、8 条（阶段0 校准不可跳过、结论先行）。
- 附录 A 固化截图 OCR 实操踩坑清单（中文包版本、转 BMP、暗色反相、bash 关沙箱直调）。
