# 阶段0：资料整理与校准（前置必做，防错核心）

目标：在动笔分析前，先把"数据从哪来、是谁、对不对"彻底搞清楚，并和用户对齐。

## 0.1 文件（Excel）按命名+表头双重识别

抖音导出的 Excel 文件名形如 `数据表现_播放量数据-11.xlsx`，但**文件名可能误导，必须以表头/实际数据列为准**。

**指标 ↔ 文件名关键词 候选映射（仅供参考，最终以表头确认）：**

| 指标 | 文件名关键词 | 易混淆陷阱 |
|------|--------------|------------|
| 播放量 | `播放量数据` | — |
| 点赞量 | `作品点赞数据` | ⚠️ 常被误读为评论 |
| 评论量 | `作品评论数据` | ⚠️ 常与点赞量级接近但意义完全不同 |
| 分享量 | `作品分享数据` | — |
| 主页访问 | `主页访问数据` | — |
| 封面点击率 | `封面点击率数据` | — |
| 作品搜索 | `作品搜索数据` | ⚠️ 易与账号搜索混 |
| 账号搜索 | `账号搜索数据` | ⚠️ 易与作品搜索混 |
| 净增粉丝 | `净增粉丝数据` | ⚠️ 易与总粉丝混 |
| 取关粉丝 | `取关粉丝数据` | — |
| 总粉丝量 | `总粉丝量数据` | ⚠️ 易与净增粉丝混 |

**操作方法**：用 Python `openpyxl`（`/Users/xdf/.workbuddy/binaries/python/envs/default/bin/python`，已装）逐一读取每个 Excel 的真实表头与首行数据，确认它到底是哪类指标。把"文件名猜的"和"表头实证的"两列并排，不一致时以表头为准。

## 0.2 截图 OCR + 内容归类

本模型直接读图会被过滤，必须用 **OCR 旁路**识别截图文字（具体实操见 `references/06-ocr-guide.md`）。识别工具优先级：

- **首选：腾讯云通用文字识别（高精度版）`tencentcloud-ocr`** —— 准召率远高于本地 tesseract，对抖音后台的暗色背景、小字号、复杂版式更稳，还能返回文字框坐标便于定位单条指标。需配置 `TENCENTCLOUD_SECRET_ID` / `TENCENTCLOUD_SECRET_KEY`。
- **兜底：本地 tesseract**（未配密钥/断网时）。流程：PIL 转 BMP → 暗色图反相 → Bash 直调（关沙箱）→ `tessdata_fast` 中文包。

> 能用腾讯云 OCR 就别用本地 tesseract，这是截图识别「更准确」的关键。若只能用本地 OCR，务必在「指标真相表」注明「关键数字以 Excel 为准」。

按 OCR 文字内容把截图归入：

| 截图类型 | 识别关键词 | 用途 |
|----------|------------|------|
| 经营看板/数据总览 | `近7日播放量` `投稿数` `互动指数` `粉丝净增` | 板块二/四 总览 |
| 关注来源 | `关注来源` `视频推荐` `我的主页` `搜索` | 板块三 |
| 单条视频 | `完播率` `2秒跳出` `5秒完播` `平均观看时长` `标题` | 板块四 三位一体 |
| 直播/引流 | `单场数据` `引流短视频` `进房` `曝光` | 板块五 |
| 粉丝画像 | `性别` `年龄` `地域` | 可选 |

## 0.3 生成「指标真相表」并请用户确认（关键闸门）

把上面两步汇成一张表，贴给用户确认。格式示例：

```
📋 数据归属核对（请确认无误后我再出报告）
┌──────────────┬─────────────────────────┬──────────────┬────────┐
│ 指标          │ 来源（文件/截图）        │ 抽样校验值    │ 状态   │
├──────────────┼─────────────────────────┼──────────────┼────────┤
│ 播放量        │ 数据表现_播放量数据-12   │ 周合计 47,342 │ ✅     │
│ 点赞量        │ 数据表现_作品点赞数据-12 │ 周合计 238    │ ✅     │
│ 评论量        │ 数据表现_作品评论数据-9  │ 周合计 13     │ ✅     │
│ 分享量        │ 数据表现_作品分享数据-10 │ 周合计 97     │ ✅     │
│ 主页访问      │ 数据表现_主页访问数据-7  │ 周合计 727    │ ✅     │
│ 封面点击率    │ 数据表现_封面点击率数据-5│ 19.28%→8.71%  │ ✅     │
│ 关注来源      │ IMG_5638 截图 OCR        │ 主页41.5%/搜索13.2% │ ✅ │
│ 单条视频×8    │ IMG_5630~5637 截图 OCR  │ 见明细        │ ✅     │
│ 直播引流×5    │ IMG_5644~5648 截图 OCR  │ 见明细        │ ✅     │
└──────────────┴─────────────────────────┴──────────────┴────────┘
⚠️ 重点核对：点赞(238) 与 评论(13) 已分别来自不同文件，未混淆。
```

**必须用 `AskUserQuestion` 或显式提问确认**："以上数据归属是否正确？有错漏请指出。" 用户确认后，把该表作为**后续分析的唯一取数字典**，正文里任何数字都回溯到这张表，中途不得另起炉灶。

> 这一步直接消灭"把点赞当评论分析"的事故：因为点赞和评论在真相表里是**两行独立记录、各自带抽样值**，且专门列出"未混淆"核对项。
