# 红线清单（务必逐条遵守）

> v3 新增 7/8；v3.1 新增 9；v3.2 新增 10（OCR 优先）。任一红线违反即视为报告不达标，必须改到全部满足再交付。

1. **阶段0 校准不可跳过**：未确认「指标真相表」前不得出分析；点赞/评论/作品搜索/账号搜索/净增粉丝/总粉丝等易混指标必须逐行核对（见 `references/00-calibration.md`）。

2. **板块二（及全报告）不碰掉粉**：不出现"掉粉 / 净增粉丝 / 取关"字眼与归因（净增粉丝仅作客观卡呈现）。

3. **趋势图不含净增粉丝**：只放播放/点赞/分享/主页访问（规范见 `references/03-chart-design.md`）。

4. **不堆数据**：每个数据都要配"为什么"（归因/客观判断），杜绝裸数字表格。

5. **客观 + 积极引导**：归因结尾一律往建设性、积极面收。

6. **单条视频双栏/三位一体**：必须有 ④⑤⑥⑦ 深层复盘；严重问题标红（框架见 `references/02-three-in-one.md`）。

7. **结论先行**：板块一必须是结论速览，不能把判断埋在最后。

8. **去 AI 味（强制过 humanizer 标准）**：成稿前必须用 `humanizer`（去AI味）技能过一遍——禁用破折号、禁用「三连/排比」结构、禁用促销腔与虚浮拔高（「显著/关键/重要/奠定了/见证」等）、不堆砌 AI 高频词、用具体细节替代空泛断言、中文用「」引号。报告里出现机器腔即视为不达标（标准见 `references/05-deai-style.md`）。

9. **亮点 / 待优化 / 优化策略 三板块定位不可混**：板块六只讲"做对了什么"（成绩单，给领导看）、板块七只讲"不足在哪"（客观短板）、板块八只讲"针对不足的解法"；板块七与板块八必须**一一对应**（N 个不足 → N 条策略），三板块都不许退化成裸数据罗列，每条都要带"为什么"与"意味着什么"。

10. **截图识别优先腾讯云高精度 OCR**：能用 `tencentcloud-ocr` 就用（准召率远高于本地 tesseract），仅在无密钥/断网时回退本地 tesseract 兜底，且须在「指标真相表」注明本地 OCR 的局限（实操见 `references/06-ocr-guide.md`）。
