# 报告板块结构（v3.2 版，共 8 大板块）

> 结构逻辑：结论先行（板块一）→ 总览与归因（板块二~三）→ 逐条深度复盘（板块四~五）→ **成绩 / 不足 / 解法 三段递进（板块六~八）**。其中板块六「亮点数据」、板块七「待优化方向」、板块八「优化策略」三者**强对应、不堆数据**：亮点是给领导看的成绩单，待优化是客观不足，优化策略是针对每条不足的一一对应解法。领导阅读路径：结论 → 总览 → 来源 → 逐条复盘 → 直播间 → 亮点 → 不足 → 解法。

> 板块四（单条视频三位一体）的详细拆解框架见 `references/02-three-in-one.md`；板块二趋势图的绘制规范见 `references/03-chart-design.md`。

---

## 板块一：核心结论速览（领导视角，结论先行）⭐

放在整份报告最前面，不堆数据，只给判断。

- **一句话总评**：本周账号处于「___」状态（如：内容质量过关但供给不稳定 / 搜索涨粉引擎启动但推荐涨粉走弱）。
- **3 个关键发现**（只放真正驱动数据走向的因素，每条一句话 + 一句支撑）：
  1. 例：播放"倒V型"——前半周有爆款，后半周接不住，说明**不是能力问题而是供给节奏问题**。
  2. 例：关注来源中"我的主页 +100%、搜索 +250%"——**主动触达路径在替代推荐成为涨粉主引擎**。
  3. 例：人均时长回落 12%——**口播类内容留人弱于切片类**，时长瓶颈在内容形态而非单条质量。
- **本周必须做的 3 件事**（动作级，非口号）：复制 07-14 切片结构 / 每条口播加"切片化"钩子 / 固定 18:00 发布。

## 板块二：核心数据总览（精简）

- **KPI 概览卡**：播放、点赞、评论、分享、主页访问、封面 CTR 的周合计 + 环比（如有前周）。净增粉丝仅作客观呈现卡、**不展开归因、不进趋势图**。
- **趋势图（Chart.js）**：绘制规范见 `references/03-chart-design.md`（双 Y 轴 / 分面、可观性有效性自检、不含净增粉丝、配色语义）。
- 去掉 v1 里 6 张"重点关注卡"的铺陈，仅保留 2-3 个最该被盯的信号，且与板块一的"关键发现"呼应。

## 板块三：关注来源与关键指标（合并原三+四）

- 来源占比条形/列表（视频推荐、我的主页、搜索、关注页、直播页、其他）+ **与上周对比分组柱状图**（蓝=本期、灰=上期估算，无前周绝对值时注明"按 上期=本期/(1+环比) 反推"）。
- 关键指标只保留**一个最该盯的**（如人均时长回落），并做内容归因，不要罗列一整屏。

## 板块五：短视频引流直播间复盘（去罗列）

- 5 场直播单场数据 + TOP 引流短视频的曝光/进房/转化。
- **必须给分析角度**，不是罗列：先给整体判断（如"直播间承接 OK，瓶颈在引流视频曝光→进房转化"），再分场次讲看点，最后给明确执行动作。
- 点出跨日复用价值（如某条切片在 07-14、07-16 两场均出现在 TOP 引流位）。

## 板块六：亮点数据（让领导看见账号上周做对了什么）⭐

**目的**：这是给领导看的"成绩单"。不是把涨了的数据全列一遍，而是**从本周数据里挑出 2-3 个真正算「亮点」的事**，讲清楚它为什么算亮点、对账号意味着什么。

**写法要求（拒绝流水账）：**
- 每个亮点必须是"有对比 / 有突破 / 有意义"的事，不能只是"XX 涨了"。
- 统一用「**现象 + 为什么算亮点（对比基线 / 环比 / 行业位置）+ 这意味着什么（对账号长期价值）**」三段式。
- 数量控制 2-3 个，多了稀释重点。亮点要"可骄傲、可复用"，不是所有正数都算亮点。
- 示例亮点（来自上周真实数据）：
  1. 07-14 直播切片播放破 1.4 万，且跨 07-14、07-16 两场直播持续出现在 TOP 引流位 → 说明"切片化"内容路线跑通，且对直播间形成**长效导流闭环**。
  2. 搜索涨粉占比 +250%、账号搜索 2,235（+153%）→ 主动搜索心智建立，账号从"被推荐刷到"转向"被主动找"。

## 板块七：待优化方向（客观指出不足，不制造焦虑）

**目的**：诚实告诉团队/领导"哪里还差"。**不是把低的数据堆一屏**，而是挑出 2-3 个真正拖后腿 / 有风险的点，说清它为什么是问题、根因在哪（呼应前面三位一体的归因）。

**写法要求：**
- 每个方向必须是"对核心目标（涨粉 / 留存 / 转化）有实质影响"的短板，不是任意低值。
- 统一用「**现象 + 为什么是问题（对目标的影响）+ 根因（呼应板块四/五的归因）**」。
- 客观中性，结尾不甩锅、不制造恐慌，指向"可改"。
- 示例待优化（来自上周真实数据）：
  1. 播放"倒V型"、后半周供给断档 → 不是能力问题，是节奏问题，致周播放总量被后半周拉低。
  2. 口播类完播率（约 3%）远低于切片（6%）、人均时长回落 12% → 内容形态留人弱，时长瓶颈在口播。

---

## 板块八：优化策略（针对待优化方向的一一对应解法）⭐

**目的**：对板块七列的每个不足，**给一条直接对应的、可落地的解决方案**。这是周报的"行动出口"，必须和待优化方向**对得上号**，不是另起炉灶喊口号。

**写法要求：**
- **强对应**：板块七有 N 个待优化点，这里就有 N 条策略，逐条对齐（推荐做成「问题 → 解法」对照表）。
- 每条策略**动作级**：谁、做什么、预期解决什么。避免"加强内容质量"这类空话。
- 示例（对应上面板块七的待优化）：
  1. 针对"供给断档" → 建选题库 + 固定 18:00 发布节奏，保证每日 ≥1 条，削峰填谷。
  2. 针对"口播留人弱" → 每条口播加"切片化"钩子（前 3 秒判断 + 中段反转），把完播率往切片水平拉。
  3. 针对"直播引流转化偏低" → 把高完播观点视频固定前置到直播前 2 小时发布，做进房承接。
- 结尾给下周 **2-3 个优先级动作**，收束全报告。
