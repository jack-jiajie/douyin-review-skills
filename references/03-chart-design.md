# 图表设计规范（趋势图 / 对比图）

图表存在的意义是「一眼看到抽象数字下的走势和异常点」，不是把数据堆进同一张图了事。生成后务必做**可观性 + 有效性自检**。

---

## 一、趋势图（板块二）

### 数据量级悬殊时的强制做法

播放量（通常数千级）与 点赞/分享/主页访问（通常几十/百级）共用单 Y 轴会导致小指标被压成一条直线，完全看不出趋势。必须采用以下方案之一：

1. **双 Y 轴法（推荐）**
   - 左轴给播放量（`beginAtZero: true`，单位次）。
   - 右轴给点赞、分享、主页访问（`beginAtZero: true`，单位次）。
   - 每个数据集显式绑定 `yAxisID`（`'y'` / `'y1'`），让两侧刻度各自适配量级。
   - legend 中标注左右轴对应关系（如 `播放量（左轴）` / `点赞（右轴）`）。
   - 示例（Chart.js）：
     ```js
     options:{responsive:true,interaction:{mode:'index',intersect:false},
       plugins:{legend:{position:'top',labels:{font:{size:12},usePointStyle:true}}},
       scales:{
         y:{type:'linear',display:true,position:'left',title:{display:true,text:'播放量'},beginAtZero:true,grid:{color:'#eef1f6'}},
         y1:{type:'linear',display:true,position:'right',title:{display:true,text:'点赞 / 分享 / 主页访问'},beginAtZero:true,grid:{drawOnChartArea:false}},
         x:{grid:{display:false}}
       }}
     ```
2. **分面小图法**：当指标超过 3 条或量级差异巨大时，拆成上下两个小图——上图专门放播放量，下图放互动与主页访问，避免互相淹没。

### 图表有效性自检（必须过）

生成后问自己：除播放量外，其他线条是否真的起伏可见？
- 如果小指标全部贴在底部 → 图表不合格，必须改成双轴或分面。
- 趋势图的目的就是让异常点（如 07-14 尖峰）和回落（如 07-18 走低）一眼可见。

### 配色（语义识别，与量级无关）

- 播放量：蓝色（实线 + 淡填充）
- 点赞：绿色
- 分享：橙色
- 主页访问：紫色

### 红线

- 趋势图**不含净增粉丝**；右轴不出现"粉丝"字样；tooltip 不得残留 `x10` 分支。

## 二、关注来源对比柱状图（板块三）

- 分组柱状图，横轴顺序：视频详情页 / 视频推荐 / 我的主页 / 直播页 / 其他 / 搜索。
- 蓝柱 = 本期占比(%)，灰柱 = 上期估算(%)。
- 无前周绝对值时，上期按 `上期 = 本期 ÷ (1 + 环比)` 反推，并在图表下方注明"仅供趋势参考"。
- Y 轴标题标注单位 `%`。

---

## 三、通用观感要求

- 图表区域给足高度（趋势图 canvas 高度建议 ≥ 230），不要挤成一条窄带。
- 坐标轴标题、单位、图例对应关系清晰，领导扫一眼能懂。
- 一张图只讲一件事，避免多主题混叠。
